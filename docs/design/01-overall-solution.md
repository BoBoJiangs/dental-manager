# 牙科管理系统整体方案

## 1. 项目定位

面向口腔诊所、牙科医生、连锁门诊集团的一体化运营管理平台，覆盖：

- 医务管理：预约、排班、接诊、病历、影像、复诊
- 运营管理：收费、财务、会员、统计
- 多端协同：PC Web、医生 H5/App、患者微信小程序
- 集团管理：统一患者池、多门诊数据协同、跨门诊排班

## 2. 核心目标

- 打通门诊接诊完整流程
- 提升医生移动办公效率
- 提升门诊经营与财务管理能力
- 建立患者运营和会员沉淀能力
- 为后续医保、税控、影像设备对接预留能力

## 3. 已确认的业务范围

### 3.1 组织形态
- 支持单诊所和连锁门诊
- 不做多租户
- 支持集团总部统一患者池
- 同一患者跨门诊共用主档案

### 3.2 财务与运营
- 完整财务流程
- 支持会员体系
- 支持短信通知
- 医保接口预留
- 发票税控接口预留

### 3.3 医疗与影像
- 支持牙位图
- 支持正畸/种植专科模板
- 支持术前术后照片对比
- 支持病例打印模板
- 电子签名为简单电子签名
- 影像先按普通文件上传处理

### 3.4 终端范围
- PC：Web 管理后台
- 医生端：H5 + App
- 患者端：微信小程序

### 3.5 权限要求
- 支持多门诊/分院
- 医生支持跨门诊排班
- 财务和院长可看全院数据
- 医生只能看自己的患者，包含历史接诊患者和当前主治患者

## 5. 组织与权限模型

### 5.1 组织结构
- org：集团
- clinic：门诊/分院
- department：科室
- room：诊室
- staff：员工

### 5.2 角色建议
- 集团管理员
- 门诊管理员
- 院长
- 前台
- 医生
- 护士/助理
- 财务
- 患者

### 5.3 数据权限建议
- 集团管理员：全集团
- 门诊管理员/院长：本门诊
- 财务：本门诊收费数据
- 医生：本人历史接诊患者 + 当前主治患者
- 前台：本门诊基础患者与预约数据

## 6. 推荐技术架构

### 6.1 架构结论
一期采用：

- 模块化单体
- 统一文件服务
- 统一短信服务
- 统一认证鉴权
- 预留外部集成适配层

### 6.2 后端技术栈
- Java 21
- Spring Boot 3
- Spring Security + JWT
- MyBatis-Plus
- MySQL 8
- Redis
- MinIO
- RabbitMQ
- Flyway
- OpenAPI

### 6.3 前端技术栈
- PC：Vue 3 + TypeScript + Element Plus
- H5/App/小程序：uni-app

## 7. 阶段规划

### 一期
- 组织权限
- 患者中心
- 预约排班
- 病历与牙位图
- 影像上传与对比
- 收费/退款/欠费/交班
- 会员基础能力
- 短信通知
- 基础统计

### 二期
- 发票税控真实对接
- 医保真实对接
- 库房采购
- 高级会员营销
- 更强经营分析

### 三期
- 影像设备 / PACS / DICOM
- 审批流
- BI 中台
- AI 辅助病历与影像

## 8. 推荐实施顺序

1. 基础底座：登录、权限、组织、字典、上传
2. 门诊核心：患者、预约、病历、牙位图、影像
3. 收费会员：收费、退款、欠费、会员、短信
4. 报表与外部集成预留
