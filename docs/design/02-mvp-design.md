# 一期 MVP 详细设计

## 1. 一期目标

一期必须打通的业务闭环：

**患者建档/预约 -> 到诊签到 -> 医生接诊 -> 病历记录 -> 牙位图 -> 影像/照片上传归档 -> 收费结算 -> 病例打印/知情同意书签字 -> 复诊预约/短信提醒 -> 经营统计**

## 3. 核心业务闭环

### 3.1 预约到接诊
1. 患者小程序预约 / 前台代预约
2. 校验医生排班和诊室资源
3. 预约成功发送短信
4. 到诊签到
5. 候诊流转
6. 医生开始接诊

### 3.2 接诊到病历归档
1. 医生查看历史病历和影像
2. 填写初诊/复诊病历
3. 标记牙位图
4. 录入诊断、治疗计划、处置内容
5. 上传口内照/面像/附件
6. 选择知情同意书模板并签名
7. 病历归档并可打印

### 3.3 收费到复诊
1. 根据治疗项目生成收费项
2. 前台/收费员开单
3. 多支付方式支付，可部分支付和挂账
4. 支持退款/冲正
5. 生成收费记录
6. 生成复诊预约和短信提醒

## 7. 一期角色与数据权限

### 集团管理员
- 查看全部门诊数据
- 管理组织、用户、统计

### 门诊管理员 / 院长
- 查看本院全部数据
- 管理本院业务配置

### 财务
- 查看本院收费、退款、欠费、日结数据

### 前台
- 预约、签到、患者登记、收费
- 查看基础患者资料

### 医生
可查看满足以下任一条件的患者：
1. 当前主治医生是本人
2. 历史接诊记录中有本人
3. 当前预约接诊医生是本人
4. 治疗计划负责人是本人
